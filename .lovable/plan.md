سأصلح منطق مساحة العمل بحيث لا تظهر رسالة “لا توجد مساحة عمل” بعد تسجيل الدخول أو عند إضافة مشروع/موقع.

## الخطة

1. **توحيد إنشاء مساحة العمل**
   - جعل منطق `ensureOwnerWorkspaceSupabase` هو المصدر الأساسي لمساحة عمل المالك.
   - التأكد أنه ينشئ/يعيد مساحة باسم `Owner Master Workspace` دائمًا عند عدم وجود واحدة.
   - ربط المساحة بمستخدم HN الحالي بدون حذف أو إعادة ضبط أي بيانات.

2. **إصلاح صفحة المشاريع**
   - منع ظهور خيار “لا يوجد مساحات عمل” كحالة نهائية إذا كان الطلب لا يزال ينشئ المساحة.
   - عند نجاح إنشاء مساحة العمل تلقائيًا، يتم اختيارها تلقائيًا في نموذج إضافة المشروع.
   - السماح بزر “إضافة” بالعمل بمجرد رجوع `workspace_id` من السيرفر.

3. **إصلاح توليد مفاتيح API**
   - إذا لم يتم تمرير `workspaceId` أو كان غير صالح، يقوم السيرفر بإنشاء/جلب مساحة العمل تلقائيًا بدل `workspace_not_found`.
   - تحديث قائمة المساحات بعد التوليد حتى تظهر المساحة الجديدة مباشرة.

4. **تغطية مسارات HN الأخرى**
   - مراجعة استخدامات: dashboard load، add site، add project، create API key، site discovery، SDK test.
   - استبدال أي اعتماد على مساحة موجودة مسبقًا بمنطق “find or create”.

5. **التحقق**
   - فحص أن صفحة المشاريع لا تعرض “لا توجد مساحة عمل” للمستخدم الجديد.
   - التأكد أن إضافة مشروع/موقع تحفظ بنجاح وتولد مفتاح API.

## ملاحظات أمان

- لن أحذف أي مشروع أو موقع أو مستخدم موجود.
- لن أعيد ضبط قاعدة البيانات.
- لن أزيل أي جداول.
- التغيير سيكون إضافة/تصحيح منطق مساحة العمل فقط.
